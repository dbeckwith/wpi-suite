Users

A user’s moderation privileges are game dependant, a game’s moderator is the creator of the game. A moderator has access to controls to begin the game, end the game, and set final estimation for the game. All other users only have access to view the game and vote on the requirements of the game.

Create a game

  • Click on Create Game button
  • Fill in the game name (a default name is provided) and game description
  • Select live game or distributed game
  • Select to use a deck or not. If you select to not use a deck, users will be able to enter any number for their estimation. If you choose to use a deck, you can hover over the available decks to see which cards are in the deck. You may also choose to create a new deck, and you will be asked to enter a name for the deck, numbers for the cards, and whether the users can select multiple cards or only a single card.
  • select requirements from the panel on the right. You have the option of creating new requirements, where you will need to enter a name, description, and choose the type of requirement. There is also an option to reload requirements from the requirements manager. This is in case someone has added requirements to the requirement manager while you are creating the game.
  • Choose if you would like the game to have a deadline or not. If yes, select a date and time that occurs in the future.
  • When you are satisfied with your new planning poker game, click the save button and you will be returned to the Game overview panel.

Edit a game

  • Before a game has been started, the creator is allowed to edit the game.
  • Select an unstarted game and click the Edit Game button.
  • The edit game panel will appear
  • Edit any information you would like to add and click save

Start a game

  • Select the unstarted game.
  • Click the start game button.

Estimate the requirement

  • Select the game that have been started.
  • Select the requirement
  • Select one card if the deck mode is single.
  • Select multiple cards if the deck mode is multiple.
  • You can see your estimate next to the submit button.
  • Deselect the cards and select others cards if you want to make a change.
  • When you are satisfied with your estimation, click the submit button.

Final Estimate

  • Only the game creator can make the final estimate.
  • Select the complete game.
  • You can see the mean and the median for the final estimate.
  • To make a change for the estimate before the game ends,select the requirement.
  • Put a estimate in the estimate field.
  • Click the save button to save the final estimate.
  • You can always make changes before the game closes or the time ends.
  • If you make the change after the first time, you have to put a note of why you want to make this change.

Close game

  • Click the game you want to close.
  • Click the close game button.
  • You will see the game status change in the complete games.

Game Tree Panel

On the game panel, the left side of the window shows a tree view of the games. There are two main groups of games: Games in Progress and Complete Games. Each of these groups have two types of games within them:

  • Games in Progress:

    • Games are marked with the icon of a blank page with a plus in the top corner have not been started yet. They can still be edited before they are started.

    Games marked with an hourglass have been started and are available to be voted on.

  • Completed Games:

    • Games Marked with a check have been completed and ready for the creator to set their final estimate.

    Games marked with a grey file cabinet icon have been closed. These games can be viewed for historical purposes but can not be changed any more.

Preferences

The preference pane lets you select how you want to receive notifications. You may select to receive email notifications, SMS notifications, or both. When you select one of the methods of notification, you will be asked to input your email or mobile number and carrier depending on which method you select.

Planning Poker

"Planning poker is a tool for estimating software development projects. It is a technique that minimizes effect of ANCHORING by asking each team member to play their estimate card such that it cannot be seen by the other players. After each player has selected a card, all cards are exposed at once."
