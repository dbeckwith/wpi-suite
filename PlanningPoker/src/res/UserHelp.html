1.1 What is Planning Poker?

Planning Poker is a tool for estimating requirements of development projects or subsets of the project. This software allows a team to reach a consensus for an estimate, an approximation of a relative amount of time needed to complete each requirement.

1.2 How Does Planning Poker Work?

A Planning Poker session is typically created when a customer or developer feels that a new specification is needed to add functionality, usability, reliability, performance, and/or supportability to the product. A moderator can begin a Planning Poker session by starting a new game and adding requirements needed for estimation. Team members are then invited to vote their estimates in each requirement. The vote is not shown to other players until the session ends. When all users have voted and the session is closed, the team is now allowed to see every vote and discuss the estimates. A consensus is reached and the final estimate is entered. This allows for fairness of answers and for each team member to independently choose their estimate without any bias and influence from other players.

1.3 Benefits of Planning Poker

Planning Poker is used to force each team member to think independently and for the entire team to view every team member’s estimates simultaneously. At times, if an individual says their number first, other members might follow their estimate. At other times, an individual might not listen to other people’s estimates and opinions because they think that their estimate is final. Planning Poker can be used in teams with difficulty expressing every member’s opinions and teams who have members with different personalities.

2.1 Quick Start

If you have used Planning Poker before but need a review on how to use it, you may want to read over the Quick Start guide.

  • Anyone can create a game.
  • Fill in the required name, description, and game type.
  • Choose to have a deck or no deck.
  • Choose to have a deadline or not.
  • Choose your requirements for estimation.
  • Edit any information before starting the game.
  • Once you save a game, you become the moderator for that game.
  • After a game starts, users can vote on each requirement either by:
    • a. typing in your estimate.
    • b. selecting one or multiple cards for your estimate value.
  • Can edit your vote for each requirement before game is completed.
  • After a game has ended, the moderator can make final estimates for each requirement.
  • Type the final estimate or stay with the default calculated estimate.
  • Click “Save.”
  • Can modify the final estimate but must add in a note stating why you changed it.
  • Select the game and click “Close Game.”

2.2 Users

A user’s moderation privileges are game dependent. A game’s Moderator is the creator of the game. A moderator has access to controls to begin the game, end the game, and set the final estimation for each requirement of the game. All other users only have access to view and vote on the requirements of the game.

2.3 Overview of the Game Tree Pane

On the game panel, the left side of the window shows a tree view of the games. There are three main groups of games: Games in Progress, Complete Games, and Closed Games. The first group has two types of games, while the second two groups only have one type of game:

  • Games in Progress:
    • Games marked with the icon of a blank page with a plus sign in the top corner have not been started yet are. They are still editable before they start.
    • Games marked with an hourglass have been started and are available to be voted on.
  • Completed Games:
    • Games in this section can no longer be voted on by users. The moderator of a completed game has the option to set final estimates for each requirement and update the Requirement Manager to reflect these requirements.
  • Closed Games:
    • Games in this section are completed, but they can no longer be changed in any way. These games can be viewed for historical purposes.

2.4 How to Create a game

  • Click on “Create Game” button.
  • Fill in the game name (a default name is provided) and game description.
  • Default game type is Distributed game. This allows users to vote on the requirement when they log in to the WPI Suite project.
  • Select to use a deck or not.
    • If you select to not use a deck, users will be able to enter any number for their estimation, unless a maximum limit is given.
    • If you select to use a deck, a default Fibonacci sequence deck is chosen.
    • You can also choose another deck from the available saved decks.
    • If you hover over the available decks, you can view the available cards in the deck.
    • You may also choose to create a new deck, and you will be asked to enter a name of the deck, numbers/estimates for each of the cards, and whether the users can select multiple cards or only a single card.
  • Select requirements from the panel on the right.
    • You have the option of creating new requirements, where you will need to enter a name, description, and choose the type of requirement.
    • There is also an option to reload requirements from the requirements manager. This is in the case when someone has added requirements to the requirement manager while you are creating the game.
  • Choose if you would like the game to have a deadline or not. If yes, select a date and time that occurs in the future.
  • When you are satisfied with your new Planning Poker game, click the “Save” button and you will be returned to the Game Overview panel.

2.5 Edit a game

  • Before a game has been started, the moderator is allowed to edit the game.
    • Select an unstarted game and click the “Edit Game” button.
    • The edit game panel will appear
    • Edit any of the following information and click “Save”: Adding/deleting requirements, deck/no deck, deadline

2.6 Start a game

  • Select the unstarted game.
  • Click the “Start Game” button.

2.7 How to Vote

  • Select the game that has already been started.
  • Select the requirement at needs to be voted.
  • Select your estimate of one card if the deck is in single mode.
  • You can select multiple cards for your estimate value if the deck is multiple mode.
  • You can see your estimate next to the submit button.
  • You can see your total estimate value next to the submit button.
  • You can deselect and select others cards if you want to make a change.
  • When you are satisfied with your estimation, click the “Submit” button.

2.8 How to End a Game

A game ends either when the deadline is up, every user has voted, or the moderator chooses to end a game at any point in time.

2.9 How to Make a Final Estimate

  • Only the game creator can make the final estimate.
  • Select the complete game.
  • You can see the mean and the median for the final estimate.
  • To make a change to the final estimate before the game closes, select the requirement.
  • Type the final estimate in the estimate field.
  • Click the “Save” button to save the final estimate.
  • You can always make changes before the game closes or the time ends.
  • If you make a change after you have a saved the final estimate, you will need to put a note of why you made this change.
  • You can always make changes to the final estimate before the game closes.

2.10 How to Close a Game

  • Select the game that you want to close.
  • Click the “Close Game” button.
  • You will see that the game has been changed to “Closed”.

2.11 Preferences

The Preferences panel lets you select how you want to receive notifications. You may select to receive email notifications, SMS notifications, or both. When you select email notifications, please input your email. When you select SMS notifications, please input your mobile number and carrier.

3.3 Some of Planning Poker’s Features

  • Mobile Application - This allows everyone to vote on a game and create a game on-the-go, so you do not need to sit in front of your computer in order to play a Planning Poker.
  • User-friendly GUI - The user is able to “fluidly” follow through all the steps from creating to ending a game.
  • Mean and Median View - To make sure that when deciding a final estimate, the team takes into account all the estimates. We have made mean and median viewable to assist you in deciding your final estimates.
  • Quick Game End - You can end a game without having to wait for everyone to finish voting.
  • Manual Final Estimate Change - To make sure that everyone knows why you have manually changed the final estimate, we added a note feature so that when you do change the final estimate, you are required to add a note.
  • Start Game Email Notification - After you created a new game, a notification will be sent to your email so even if someone has not opened the Planning Poker module, they will still know that a game has begun.
  • Easy Step-by-Step Help - Our step-by-step help documentation will help save you a lot of time if you are a beginning Planning Poker user or just want to review how Planning Poker works with our Quick Start guide.
